## Tasks

### Task 1: Create `.nvmrc` and bump Node/pnpm versions

**Description:** Create `.nvmrc` pinned to Node 24. In `package.json`, bump `engines.node` from `>=18.0.0` to `>=24.0.0` and `packageManager` from `pnpm@10.28.2` to `pnpm@10.29.3`.
**Files:**

- `.nvmrc` ‚Äî create
- `package.json` ‚Äî modify

**Code example:**

`.nvmrc`:

```
24
```

`package.json` changes:

```json
"packageManager": "pnpm@10.29.3",
"engines": {
  "node": ">=24.0.0"
}
```

**Done when:** `.nvmrc` contains `24`, `package.json` has `>=24.0.0` engine and `pnpm@10.29.3` packageManager
**Commit:** "Bump Node to 24 and pnpm to 10.29.3"

---

### Task 2: Upgrade Zod to v4 and remove redundant typescript-eslint deps

**Description:** In `package.json`, change `zod` from `^3.24.0` to `^4.0.0`. Remove `@typescript-eslint/eslint-plugin` and `@typescript-eslint/parser` from devDependencies. Run `pnpm install` to regenerate the lock file. Verify with `pnpm typecheck`, `pnpm lint`, and `pnpm build`.
**Files:**

- `package.json` ‚Äî modify
- `pnpm-lock.yaml` ‚Äî regenerated by `pnpm install`

**Done when:**

- `pnpm install` completes with no unmet peer dependency warnings
- `pnpm typecheck` passes
- `pnpm lint` passes
- `pnpm build` succeeds
- No `@typescript-eslint/eslint-plugin` or `@typescript-eslint/parser` in `package.json` devDependencies

**Commit:** "Upgrade Zod to v4 and remove redundant typescript-eslint deps"

---
