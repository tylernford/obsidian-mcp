## Overview

**What:** Address all 13 issues identified in the post-TypeScript refactor QA review (`docs/notes/typescript-review.md`).

**Why:** The codebase has non-null assertions that mask potential runtime crashes, unhandled exceptions, and incorrect config scoping. These undermine the type safety that the TypeScript migration was meant to provide.

**Type:** Bugfix

## Design Decisions

### 1. Widening `apiKey` type vs. swapping `!` for `as string`

**Options considered:**

1. `as string` — minimal change, still a type lie but less hidden
2. `string | undefined` — honest type, let the constructor validate

**Decision:** Option 2. The constructor already has the runtime check (`if (!apiKey) throw`). Widening the type makes the type system reflect reality instead of lying about it.

### 2. `ApiResponse<T>` — generic only vs. generic + Zod runtime validation

**Options considered:**

1. Generic only (`ApiResponse<T = unknown>`) — ergonomic improvement, no runtime cost
2. Generic + optional Zod schema parameter — runtime validation of response shapes

**Decision:** Option 1. The Obsidian Local REST API is a trusted local dependency with stable, predictable responses. Runtime validation of a trusted source is over-engineering. The generic improves call-site ergonomics and makes expected shapes visible in type signatures.

### 3. HTTPS support — include vs. drop

**Options considered:**

1. Add `protocol` config field (`"http" | "https"`, default `"http"`)
2. Drop — keep hardcoded `http://`

**Decision:** Option 2. The Obsidian Local REST API runs on localhost. HTTPS for localhost provides no security benefit and adds config surface (env var, documentation) for a use case nobody is asking for.

### 4. `ApiResponse<T>` call-site updates — now vs. later

**Options considered:**

1. Update call sites that use `as` casts to use the generic type parameter instead
2. Leave call sites as-is since `T = unknown` default is backwards-compatible

**Decision:** Option 1. Only two call sites have `as` casts (both in `metadata.ts`). Small change, better to clean up now.
