# Upgrade Dependencies

**Created:** 2026-02-15
**Status:** Design
**Implementation Plan Doc:** docs/implementation-plans/2026-02-15-1556-upgrade-dependencies.md

## Overview

**What:** Upgrade all project dependencies to their latest compatible versions and clean up redundant packages.

**Why:** The project was scaffolded with conservative version pins (Node 18, Zod 3) and accumulated redundant `@typescript-eslint/*` direct dependencies. A fresh pass brings everything current and removes unnecessary packages.

**Type:** Enhancement

---

## Requirements

### Must Have

- [ ] Bump `engines.node` from `>=18.0.0` to `>=24.0.0`
- [ ] Bump `packageManager` from `pnpm@10.28.2` to `pnpm@10.29.3`
- [ ] Upgrade `zod` from `^3.24.0` to `^4.0.0`

- Prettier config changes

## Design Decisions

### Keep ESLint at v9

**Options considered:**

1. Upgrade ESLint to v10 ŌĆö would require `typescript-eslint` to ship ESLint 10 support (not yet available as of 2026-02-15, latest is v8.55.0)
2. Stay on ESLint v9 ŌĆö fully compatible with current `typescript-eslint@8`

**Decision:** Stay on ESLint 9. The upgrade is blocked upstream. Revisit when `typescript-eslint` adds `eslint ^10.0.0` to its peer dependencies.

### Zod 3 ŌåÆ 4

**Options considered:**

1. Upgrade now ŌĆö the MCP SDK (`@modelcontextprotocol/sdk`) already accepts `zod ^3.25 || ^4.0`, and the project only uses basic Zod primitives (`z.string()`, `z.enum()`, `z.array()`, `.describe()`, `.optional()`) which are unchanged in Zod 4
2. Defer to a separate effort ŌĆö lower risk but unnecessary given the minimal surface area

**Decision:** Upgrade now. Usage is limited to 7 files with only basic schema builders. Expected to be a zero-code-change upgrade.

### Remove redundant @typescript-eslint direct deps

**Options considered:**

1. Keep them ŌĆö no harm, just noise
2. Remove them ŌĆö the unified `typescript-eslint` package re-exports the plugin and parser internally; they remain installed transitively

**Decision:** Remove them. Reduces confusion about which packages are actually needed. The unified package handles everything.
